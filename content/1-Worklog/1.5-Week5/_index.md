### Week 5 Objectives:
* Master the mechanisms of encrypting data at rest and data in transit using the AWS KMS service.
* Learn about comprehensive user management, authentication, and authorization solutions with Amazon Cognito.
* Enhance skills in writing IAM Policies (especially Policy Conditions and PassRole) to securely grant Execution Roles for Serverless services.

### Tasks carried out this week:

| Day | Task | Start Date | Completion Date | Reference Material (AWS Study Group Link) |
| :--- | :--- | :--- | :--- | :--- |
| **Monday**<br>(Day 1) | - Learn about AWS KMS key management service:<br>&emsp; + Differentiate between Customer Managed Keys (CMK) and AWS Managed Keys.<br>&emsp; + Concepts: Symmetric/Asymmetric keys, Key Rotation.<br>&emsp; + Envelope Encryption mechanism. | 18/05/2026 | 18/05/2026 | <https://cloudjourney.awsstudygroup.com/> |
| **Tuesday**<br>(Day 2) | - Learn about Amazon Cognito user management service:<br>&emsp; + Differentiate between Cognito User Pools (Authentication) and Identity Pools (Authorization).<br>&emsp; + Understand the operating mechanism of the OAuth 2.0 standard and JWT (JSON Web Tokens). | 19/05/2026 | 19/05/2026 | <https://cloudjourney.awsstudygroup.com/> |
| **Wednesday**<br>(Day 3) | - Deeply research AWS IAM Policies:<br>&emsp; + Use Policy Conditions to restrict access by IP or time.<br>&emsp; + The **iam:PassRole** concept when configuring automated services. | 20/05/2026 | 20/05/2026 | <https://cloudjourney.awsstudygroup.com/> |
| **Thursday**<br>(Day 4) | - **KMS & S3 Practice:**<br>&emsp; + Create a Customer Managed Key named **fcaj-idp-kms-key**.<br>&emsp; + Configure the S3 Bucket (**fcaj-idp-docs...**) to automatically encrypt all uploaded files using this KMS key. | 21/05/2026 | 21/05/2026 | <https://cloudjourney.awsstudygroup.com/> |
| **Friday**<br>(Day 5) | - **Cognito Practice:**<br>&emsp; + Initialize **fcaj-idp-user-pool** to manage system login user accounts.<br>&emsp; + Configure App Client and enable Cognito Hosted UI.<br>&emsp; + Test the Sign-up and Sign-in flow on the available web interface. | 22/05/2026 | 22/05/2026 | <https://cloudjourney.awsstudygroup.com/> |

### Achievements:
* Clearly understood the importance of encrypting sensitive data and successfully configured AWS KMS manually to protect documents stored on Amazon S3.
* Built a solid user authentication foundation using Amazon Cognito User Pool, ready to secure API endpoints (API Gateway) for the upcoming project.
* Capable of visually testing the login flow via Cognito Hosted UI without needing to write complex Frontend code.
* Practiced strict security thinking (Zero Trust) by applying advanced Policy conditions in AWS IAM.
